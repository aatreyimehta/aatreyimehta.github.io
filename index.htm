Project Outline:

Globally, the amount of digital data produced is increasing at an exponential rate. While the sources of this data, commonly known as Big Data, range from mobile services to cyber physical systems and beyond, the one constant is that it will continue to rise at an exponential rate in the foreseeable future. With the ever-increasing volume of data, the methods to store and process it are also becoming vital. In this project, the major focus will be on parallel processing as well as batch processing on Big Data. On one hand, the parallel processing platforms are a considerable solution for big data mining. The objective of parallel processing is to break down a huge problem into smaller chunks, each of which is handled by a single processor. Furthermore, these procedures are carried out in a distributed and parallel manner. Whereas, Batch processing is useful when you don't require real-time analytics and it's more essential to analyze huge amounts of data in order to gain more thorough insights than it is to acquire quick analytics results. Batch processing is used for applications that are frequently used and can be run with little human intervention.
I intend to portray a comparative analysis based on the formerly stated ideas related to parallel and batch processing on Big Data.
